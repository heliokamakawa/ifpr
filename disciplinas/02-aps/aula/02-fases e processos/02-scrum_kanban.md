# O que é Scrum?

**Scrum é um framework (não uma metodologia completa).**  
Ele é propositalmente incompleto e define uma estrutura básica para ciclos de trabalho.

## Estrutura Principal

- **Sprint (2 a 4 semanas)** → ciclo fechado de trabalho (timebox)
- **Sprint Planning** → planejamento do ciclo
- **Daily Scrum** → acompanhamento diário
- **Sprint Review** → inspeção do produto
- **Retrospectiva** → melhoria do processo

## Papéis

- **Scrum Master** → cuida do processo
- **Product Owner** → priorização e visão de negócio
- **Time** → desenvolvimento

## Pilares

- Transparência  
- Inspeção  
- Adaptação  

## Pontos Fortes

- Ciclos curtos de feedback
- Entregas frequentes
- Foco em produto
- Estímulo à melhoria contínua

## Limitações

Pode apresentar dificuldades em ambientes com:
- Demandas imprevisíveis
- Manutenção com urgências variadas
- Diferentes classes de prioridade simultâneas

# O que é Kanban?

**Kanban é um método de gestão de mudanças e melhoria evolutiva.**

Ele não substitui seu processo atual — ele o evolui.

## Fundamentos

- Visualização do fluxo de trabalho
- Gestão de capacidade
- Evolução contínua
- Respeito ao processo existente

## Características

- Altamente flexível
- Aplicável em qualquer área (TI, jurídico, marketing, RH etc.)
- Não impõe papéis fixos
- Não exige rituais específicos
- Trabalha com fluxo contínuo

## Foco Principal

- Tornar o trabalho visível
- Identificar gargalos
- Melhorar gradualmente o sistema

---

# Scrum vs Kanban — A Verdade Estratégica

A pergunta **“qual é melhor?”** é inadequada.

- Kanban não substitui Scrum.
- Kanban pode melhorar um Scrum existente.
- Se o Scrum funciona bem, não há motivo para trocar.
- Kanban é especialmente útil quando há dores ou ineficiências.

---

# Diferença Estrutural

| Scrum | Kanban |
|--------|----------|
| Trabalha com lotes fechados (Sprint) | Trabalha com fluxo contínuo |
| Unidade de capacidade = Sprint | Unidade de capacidade = fluxo |
| Planejamento fixo por ciclo | Reabastecimento contínuo |
| Menor flexibilidade interna | Alta adaptação a demandas variadas |

---

# Insight Importante

- Toda organização possui um fluxo.
- Se existe entrega, existe fluxo.
- Se existe fluxo, é possível aplicar Kanban.

Scrum é um tipo de fluxo.  
Cascata é um tipo de fluxo.  
Até ambientes caóticos possuem fluxo.

---

# Quando Mudar?

Mudança só ocorre quando existem:

1. Insatisfação real
2. Reflexão estruturada
3. Liderança ativa

Sem dor, não há evolução.

---

# Conclusão

- **Scrum** é excelente para ciclos estruturados e foco em produto.
- **Kanban** é poderoso para evolução contínua e ambientes dinâmicos.
- Não é uma disputa.
- É uma decisão contextual.

> A pergunta correta não é “Scrum ou Kanban?”  
> A pergunta correta é: **qual abordagem melhora nosso sistema e atende melhor nosso cliente?**
