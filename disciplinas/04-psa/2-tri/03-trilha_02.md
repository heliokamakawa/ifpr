# Trilha 2 - Projeto curto com agente de IA

## Objetivo

Desenvolver um projeto pequeno com agente de IA e melhorar o prompt ao longo do processo.

A dupla deve aplicar um prompt inicial, avaliar o resultado e o custo em tokens, depois melhorar o prompt usando tecnicas de engenharia de prompt e contexto tecnico mais especifico.

Depois da melhoria, a dupla deve comparar as versoes para mostrar se houve ganho de qualidade e qual foi o custo em tokens.

## Projeto

O projeto deve usar:
- SOLID;
- Padrao de projeto;
- agente de IA como apoio.

O escopo precisa caber em pouco tempo.

Exemplos:
Strategy, Observer, Factory, Adapter, DTO, service, repository para fazer:
- calculo de frete usando;
- sistema simples de notificações;

## Engenharia de prompt

A dupla deve buscar uma metodologia de engenharia de prompt e seguir essa metodologia.

Se nao encontrar uma metodologia adequada, pode desenvolver uma propria, mas deve explicar:

- quais regras foram usadas;
- por que essas regras foram escolhidas;
- como elas melhoraram o prompt.

O ideal e buscar uma metodologia existente e usar como referencia.

## Como fazer

A dupla deve executar o mesmo escopo pelo menos **duas vezes**.

### Tentativa 1 - Prompt inicial

Use um prompt simples.

Registre:

- prompt usado;
- ferramenta ou modelo;
- resultado gerado;
- tokens de entrada;
- tokens de saida;
- avaliacao da qualidade.

### Tentativa 2 - Prompt melhorado

Melhore o prompt usando:

- tecnica de engenharia de prompt;
- contexto tecnico mais especifico;
- linguagem;
- escopo;
- padrao de projeto;
- restricoes do que deve e nao deve ser feito.

Registre novamente:

- prompt usado;
- resultado gerado;
- tokens de entrada;
- tokens de saida;
- avaliacao da qualidade;
- comparacao com a tentativa 1.

### Tentativa 3, se der tempo

Refine novamente o prompt com base nos problemas anteriores.

## Avaliacao da qualidade

A dupla deve observar:

- o codigo funcionou?
- o escopo foi respeitado?
- o padrao de projeto foi bem aplicado?
- as classes tinham responsabilidades claras?
- houve excesso de codigo?
- o prompt melhorado aumentou ou reduziu o custo em tokens?
- o aumento de tokens gerou melhoria real de qualidade?
- a dupla entendeu o codigo gerado?
